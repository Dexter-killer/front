## 📋 Обзор проекта
Данный проект представляет собой полнофункциональное веб-приложение для управления личными заметками и планирования напоминаний. Приложение построено на базе современных технологий **PWA (Progressive Web App)**, что позволяет ему работать в офлайн-режиме, устанавливаться на мобильные устройства и получать Push-уведомления даже при закрытой вкладке браузера.

## 🛠 Технологический стек
*   **Frontend:** HTML5 (App Shell Architecture), CSS (Chota framework), Vanilla JavaScript.
*   **Backend:** Node.js, Express.
*   **Real-time:** Socket.io (для мгновенной синхронизации между клиентами).
*   **PWA:** Service Workers, Web App Manifest, Cache API.
*   **Notifications:** Push API, Web-Push (VAPID), Notification API.
*   **Security:** HTTPS (SSL/TLS самоподписанные сертификаты).

## 🚀 Реализованный функционал (по этапам разработки)

### 1. Архитектура и Офлайн-режим (№13, №15)
Реализована архитектура **App Shell**, которая обеспечивает мгновенную загрузку интерфейса. 
*   **Service Worker:** Скрипт `sw.js` управляет кэшированием статических ресурсов (интерфейс, стили, скрипты).
*   **Стратегия кэширования:** Для статики используется "Cache First", для динамического контента (страницы Home/About) — "Network First" с фолбеком на кэш.
*   **LocalStorage:** Данные пользователя сохраняются локально, что позволяет просматривать и добавлять заметки без доступа к сети.

### 2. Установка и Мобильность (№14)
Создан **Web App Manifest**, позволяющий устанавливать приложение как нативное:
*   Настроены иконки всех стандартных размеров (от 16x16 до 512x512).
*   Определены цвета темы и фона для корректного отображения Splash Screen.
*   Приложение работает в режиме `standalone` (без адресной строки браузера).

### 3. Синхронизация в реальном времени (№16)
Интегрирована библиотека **Socket.io**. При добавлении заметки одним пользователем, все остальные подключенные клиенты получают мгновенное уведомление в интерфейсе без перезагрузки страницы.

### 4. Система Push-уведомлений и Напоминаний (№16, №17)
Реализована серверная логика планирования уведомлений:
*   **VAPID:** Использование протокола безопасной идентификации сервера уведомлений.
*   **Reminders:** Возможность установить точную дату и время напоминания.
*   **Snooze (Отложить):** В само Push-уведомление встроена интерактивная кнопка «Отложить на 5 минут». При клике Service Worker отправляет запрос на сервер для перепланирования уведомления.

### 5. Безопасность (№15)
Приложение поддерживает работу по протоколам **HTTP (порт 3000)** и **HTTPS (порт 3001)**. Использование HTTPS является обязательным условием для работы Push API в современных браузерах.

## ⚙️ Инструкция по запуску

1.  **Установка зависимостей:**
    ```bash
    npm install
    ```
2.  **Запуск сервера:**
    ```bash
    node server.js
    ```
3.  **Доступ к приложению:**
    *   **HTTP:** [http://localhost:3000](http://localhost:3000) (рекомендуется для базовой отладки)
    *   **HTTPS:** [https://localhost:3001](https://localhost:3001) (необходимо для работы уведомлений)

> **Примечание:** При первом входе через HTTPS браузер выдаст предупреждение о самоподписанном сертификате. Необходимо нажать "Дополнительно" -> "Перейти на сайт localhost (небезопасно)".

## 🧪 Тестирование
1.  Откройте две вкладки браузера. Добавьте заметку в одной — она мгновенно появится в другой (Socket.io).
2.  Включите уведомления, установите напоминание на 1 минуту вперед и закройте вкладку. Вы получите системное уведомление.
3.  Отключите интернет в DevTools (Network -> Offline) и обновите страницу. Приложение продолжит работать (Service Worker).
