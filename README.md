## 🌟 Введение
Данный проект представляет собой результат выполнения цикла практических работ (№13–18). Основная цель — создание современного, отказоустойчивого веб-приложения, которое объединяет в себе лучшие черты нативных мобильных приложений и традиционных веб-сайтов. 

Использование технологии **Progressive Web App (PWA)** позволяет пользователю взаимодействовать с приложением в условиях нестабильного интернет-соединения или его полного отсутствия, устанавливать его на домашний экран устройства и получать персонализированные Push-уведомления.

## 🛠 Техническая архитектура и стек технологий

### Frontend (Клиентская часть)
*   **App Shell Architecture:** Разделение приложения на неизменяемую "оболочку" (HTML/CSS/JS) и динамически подгружаемый контент. Это минимизирует объем передаваемых данных и ускоряет рендеринг.
*   **Vanilla JavaScript (ES6+):** Весь функционал реализован без тяжелых фреймворков для демонстрации чистого взаимодействия с браузерными API.
*   **Chota CSS:** Минималистичный CSS-фреймворк для быстрого создания адаптивной и доступной верстки.
*   **Socket.io Client:** Обеспечивает двустороннюю связь с сервером.

### Backend (Серверная часть)
*   **Node.js & Express:** Обработка статических файлов, API-запросов и маршрутизация.
*   **Socket.io Server:** Управление активными соединениями и рассылка широковещательных (broadcast) сообщений.
*   **Web-Push (VAPID):** Библиотека для шифрования и отправки Push-пакетов через серверы поставщиков уведомлений (Google FCM, Mozilla Autopush).
*   **HTTPS/SSL:** Реализация защищенного канала передачи данных с использованием модулей `fs` и `https`.

## 📂 Детальное описание этапов разработки (Практические работы №13–18)

### Практическая работа №13: Внедрение Service Worker
Был разработан сценарий `sw.js`, работающий в фоновом режиме отдельно от основного потока браузера.
*   **Жизненный цикл:** Реализованы события `install` (первичное кэширование), `activate` (очистка устаревшего кэша) и `fetch` (перехват сетевых запросов).
*   **Offline First:** Приложение проверяет наличие ресурсов в `Cache Storage` перед обращением к сети, что гарантирует работоспособность интерфейса без интернета.

### Практическая работа №14: Манифест и установка
Создана конфигурация `manifest.json`, соответствующая стандартам W3C.
*   **Standalone Mode:** Настроен запуск приложения в отдельном окне без элементов интерфейса браузера.
*   **Branding:** Подготовлен набор иконок (16x16 – 512x512) и определены `theme_color` (цвет статус-бара) и `background_color` (цвет загрузочного экрана).

### Практическая работа №15: App Shell и Безопасность (HTTPS)
*   **HTTPS:** Для обеспечения работы Service Worker и безопасности данных были сгенерированы SSL-сертификаты. Сервер настроен на одновременную поддержку защищенного (порт 3001) и стандартного (порт 3000) соединений.
*   **Динамический контент:** Реализована подгрузка фрагментов страниц (`home.html`, `about.html`) через `fetch` с последующим кэшированием по стратегии "Network First".

### Практическая работа №16: Реальное время (WebSocket) и Push API
*   **WebSocket:** Реализована синхронизация заметок. Когда пользователь «А» добавляет заметку, пользователь «Б» мгновенно видит её благодаря `socket.emit` и `socket.broadcast`.
*   **Push-подписка:** Реализован механизм получения `Subscription Object` от браузера и его передача на сервер для хранения.

### Практическая работа №17: Интеллектуальные напоминания
Разработана система отложенных задач на стороне сервера.
*   **Серверный планировщик:** При получении события `newReminder`, сервер вычисляет задержку до наступления события и устанавливает таймер.
*   **Интерактивные уведомления:** В Push-сообщение добавлены кнопки действий (`actions`). Одной из ключевых функций стала кнопка **«Отложить на 5 минут»**, которая позволяет пользователю взаимодействовать с сервером прямо из центра уведомлений ОС.

### Практическая работа №18: Финализация и README
Итоговая сборка проекта, проведение нагрузочного тестирования WebSocket-соединений и оформление проектной документации.

## 🏗 Как запустить проект

1.  **Клонирование и установка:**
    ```bash
    git clone https://github.com/Dexter-killer/front.git
    cd front
    npm install
    ```
2.  **Запуск серверов:**
    ```bash
    node server.js
    ```
3.  **Использование:**
    *   Для полноценного тестирования Push-уведомлений используйте: `https://localhost:3001`.
    *   При появлении предупреждения о сертификате в браузере выберите: "Дополнительно" -> "Перейти на сайт".

## 🧪 Сценарии тестирования
1.  **Офлайн:** Откройте приложение, выключите интернет, добавьте заметку. Она сохранится в LocalStorage. Включите интернет — приложение готово к работе.
2.  **Синхронизация:** Откройте две разные вкладки (или два разных браузера). Добавление заметки в одной мгновенно отобразится в другой.
3.  **Напоминание:** Установите время на 1 минуту вперед. Закройте вкладку браузера. Через минуту вы получите системное Push-уведомление с кнопкой "Отложить".
