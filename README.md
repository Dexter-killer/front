## 📝 1. Введение и концепция проекта
Данный проект является комплексной реализацией современных веб-технологий, направленных на создание **Progressive Web Application (PWA)**. Основная идея заключается в преодолении разрыва между веб-сайтами и нативными приложениями. 

Приложение «Заметки» предоставляет пользователю возможность не только сохранять текстовую информацию, но и настраивать интеллектуальную систему напоминаний, которая работает в фоновом режиме. Проект демонстрирует глубокую интеграцию клиентской и серверной частей, использование двусторонних каналов связи (WebSockets) и систем системных уведомлений ОС.

## 🛠 2. Подробный технологический стек

### 🌐 Frontend (Клиентская часть)
*   **Архитектура:** App Shell (Оболочка приложения). Весь интерфейс загружается один раз и кэшируется, а контент меняется динамически без перезагрузки страницы.
*   **Интерфейс:** 
    *   **HTML5 & CSS3:** Адаптивная верстка, поддерживающая мобильные и десктопные устройства.
    *   **Chota CSS:** Легковесный (3кб) CSS-фреймворк, обеспечивающий современный вид без перегрузки проекта.
*   **Скрипты:** 
    *   **Native DOM API:** Манипуляция элементами страницы без использования jQuery или реактивных фреймворков.
    *   **Fetch API:** Асинхронные запросы к серверу для получения динамического контента и подписки на Push.
    *   **Socket.io Client:** Обеспечение связи с сервером по протоколу WebSocket.
    *   **Service Worker API:** Управление кэшем и перехват сетевых событий.

### ⚙️ Backend (Серверная часть)
*   **Платформа:** Node.js (v18+).
*   **Фреймворк:** Express.js (управление маршрутизацией и статическими файлами).
*   **Middleware:** 
    *   `body-parser`: Обработка JSON-данных в теле запросов.
    *   `cors`: Настройка разрешений для кросс-доменных запросов (важно для Socket.io).
*   **Связь:** Socket.io (Server-side) — обработка событий синхронизации и напоминаний.
*   **Уведомления:** `web-push` — реализация протокола VAPID для защищенной отправки Push-пакетов.
*   **Безопасность:** Реализация HTTPS-сервера с использованием TLS/SSL сертификатов.

## 📂 3. Глубокий разбор реализованных модулей

### 3.1. Service Worker (`sw.js`)
Service Worker — это "прокси-сервер" на стороне клиента. В проекте реализована продвинутая логика:
*   **Событие `install`:** Приложение скачивает все необходимые файлы (HTML, CSS, JS, иконки) и сохраняет их в `Cache Storage`.
*   **Событие `activate`:** Происходит автоматическая ротация кэша (удаление старых версий при обновлении приложения).
*   **Событие `fetch`:** Перехват всех сетевых запросов. Применяется стратегия **"Stale-while-revalidate"** — пользователь мгновенно получает данные из кэша, а в фоне приложение проверяет наличие обновлений на сервере.
*   **Событие `push`:** Обработка входящих уведомлений. SW анализирует JSON-объект уведомления и вызывает `showNotification` с кнопками действий.
*   **Событие `notificationclick`:** Логика обработки кликов. Если нажата кнопка «Отложить», SW делает POST-запрос к API сервера.

### 3.2. Web App Manifest (`manifest.json`)
Файл конфигурации, который превращает сайт в приложение:
*   **`display: standalone`**: Удаляет адресную строку браузера, делая сайт неотличимым от нативного приложения.
*   **`icons`**: Набор из 8 различных размеров иконок для поддержки всех типов экранов (Android, iOS, Windows).
*   **`theme_color`**: Задает цвет верхней панели браузера в цвет приложения.

### 3.3. Логика синхронизации (Socket.io)
В приложении реализован паттерн **Event-Driven Architecture**:
1.  Клиент отправляет событие `newTask`.
2.  Сервер получает его и транслирует (`broadcast`) всем остальным подключенным пользователям.
3.  Другие пользователи видят всплывающее уведомление (Toast) о новой заметке без обновления страницы.

## 📡 4. Описание API Эндпоинтов

| Метод | Путь | Описание |
| :--- | :--- | :--- |
| `POST` | `/subscribe` | Регистрация подписки пользователя на Push-уведомления. |
| `POST` | `/unsubscribe` | Удаление подписки (отписка). |
| `POST` | `/snooze` | Перенос времени напоминания на 5 минут вперед (вызывается из уведомления). |
| `GET` | `/content/*` | Получение фрагментов HTML-кода для архитектуры App Shell. |

## 🔒 5. Безопасность и HTTPS
Для работы Push-уведомлений и Service Worker в проекте реализован HTTPS:
*   **Сертификаты:** Используются файлы `cert.pem` и `key.pem`.
*   **Порты:** 
    *   **3000 (HTTP):** Для быстрой отладки и тестирования базовой логики.
    *   **3001 (HTTPS):** Для полноценного тестирования PWA-функций.

## 🛠 6. Инструкция по развертыванию и эксплуатации

### Предварительные требования
*   Установленный **Node.js** (версии 18 и выше).
*   Браузер на движке Chromium (Chrome, Edge) для корректной работы Push API.

### Порядок запуска
1.  **Склонируйте репозиторий:**
    ```bash
    git clone https://github.com/Dexter-killer/front.git
    cd front
    ```
2.  **Установите пакеты:**
    ```bash
    npm install
    ```
3.  **Запустите серверную часть:**
    ```bash
    node server.js
    ```
4.  **Доступ:**
    *   Откройте [https://localhost:3001](https://localhost:3001).
    *   **ВАЖНО:** Так как сертификат самоподписанный, браузер покажет предупреждение. Нажмите **«Дополнительно»** -> **«Перейти на сайт (небезопасно)»**.

## 🐞 7. Устранение неполадок (Troubleshooting)

1.  **Ошибка `SecurityError: Failed to register a ServiceWorker`**:
    *   *Причина:* Попытка регистрации на незащищенном соединении или проблема с доверием к сертификату.
    *   *Решение:* Используйте `localhost` (браузеры доверяют ему даже по HTTP) или подтвердите исключение безопасности в браузере для HTTPS.
2.  **Уведомления не приходят**:
    *   *Причина:* Не получено разрешение в браузере или ОС заблокировала уведомления (режим «Не беспокоить»).
    *   *Решение:* Проверьте настройки разрешений в адресной строке (значок замка) и настройки уведомлений Windows.
3.  **Ошибка `EADDRINUSE` при запуске**:
    *   *Причина:* Порты 3000 или 3001 заняты другой программой или предыдущим запуском сервера.
    *   *Решение:* Завершите процесс `node` в диспетчере задач или перезагрузите терминал.
